---
title: "Chatbot settings"
description: "Configure your chatbot's name, API keys, team access, webhooks, message analysis, and data import/export."
---

The settings page is where you configure your chatbot. Manage API keys, control which messages get analyzed, set up webhooks, and manage team access — all from one place.

![Chatbot settings](/images/chatbot-settings.png)

<Tabs>
  <Tab title="General">
    - **Chatbot name** — Edit the display name that appears throughout the dashboard. Click save after changing.
    - **Chatbot ID** — A unique identifier for this chatbot. Click to copy.
    - **Created** — When this chatbot was created.
    - **Message analysis toggles** — Two switches control which messages get analyzed:
      - **User message analysis** — When enabled, OpenBat extracts sentiment, intent, topics, and flags from user messages.
      - **Assistant message analysis** — When enabled, OpenBat detects behavior, outcomes, and quality flags from assistant messages.

    <Info>
      Disabling analysis for a message type reduces costs but removes those signals from your dashboard and conversation views.
    </Info>

    - **Danger zone** — Permanently delete this chatbot and all its conversations, messages, and analytics data.

    <Warning>
      Deleting a chatbot is irreversible. All conversations, messages, and analytics data will be permanently removed.
    </Warning>
  </Tab>

  <Tab title="Company info">
    Associate company details with this chatbot for context and organization.
  </Tab>

  <Tab title="API keys">
    - **View your API key** — The key prefix is displayed for identification. The full key is never shown after creation for security.
    - **Rotate key** — Generate a new API key. The old key is invalidated immediately.

    <Warning>
      Rotating your API key immediately invalidates the old key. Update your SDK configuration before rotating to avoid data loss.
    </Warning>
  </Tab>

  <Tab title="Members">
    Manage who has access to this specific chatbot. Add or remove team members at the chatbot level, separate from organization-level membership.
  </Tab>

  <Tab title="Webhooks">
    Configure webhook endpoints for automated notifications:

    - **Slack webhooks** — Send alerts to Slack channels.
    - **HTTP webhooks** — Send POST requests to any URL.

    Each webhook shows its name, URL, type (Slack or HTTP), and enabled/disabled status.
  </Tab>

  <Tab title="Import/Export">
    - **Import data** — Bring in conversation data from external sources.
    - **Export data** — Download your conversation and analytics data.
  </Tab>
</Tabs>
