Workflows let you automate responses to conversation analysis. Set up triggers that fire when specific conditions are met — negative sentiment, churn risk detected, hallucination flagged — and route alerts to Slack, email, or any HTTP webhook.

![Workflows page](/images/workflows.png)

## The workflows page

Three tabs organize workflow management:

- **Workflows** — List of all configured workflows with their status
- **Runs** — Execution history showing when workflows triggered
- **Templates** — Pre-built workflow recipes to get started fast

## Creating a workflow

<Steps>
  <Step title="Navigate to workflows">
    Navigate to the **Workflows** page from the sidebar.
  </Step>

  <Step title="Create a new workflow">
    Click **+ Create workflow**.
  </Step>

  <Step title="Set trigger conditions">
    Choose what triggers the workflow. For example, a sentiment score below a threshold, a specific behavior detected, or a flag like churn risk.
  </Step>

  <Step title="Configure actions">
    Choose where to send the alert:

    - **Slack** — Post a message to a Slack channel
    - **Email** — Send a notification email
    - **HTTP webhook** — Send a POST request to any URL
  </Step>

  <Step title="Save and activate">
    Save and activate the workflow. It starts monitoring new conversations immediately.
  </Step>
</Steps>

## Example workflows

<AccordionGroup>
  <Accordion title="Churn risk alert">
    **Trigger:** user flag = `churn_risk`

    **Action:** Slack message to `#customer-success` with user details.
  </Accordion>

  <Accordion title="Hallucination detected">
    **Trigger:** assistant behavior = `hallucinating`

    **Action:** Slack message to `#engineering` with conversation link.
  </Accordion>

  <Accordion title="Negative sentiment spike">
    **Trigger:** sentiment score < -0.5

    **Action:** Email to product manager.
  </Accordion>

  <Accordion title="Buying signal">
    **Trigger:** user flag = `buying_signal`

    **Action:** Slack message to `#sales` with user and org info.
  </Accordion>

  <Accordion title="Unresolved conversation">
    **Trigger:** resolution outcome = `deflected`

    **Action:** HTTP webhook to ticketing system.
  </Accordion>
</AccordionGroup>

## Runs tab

The Runs tab shows the execution history for all workflows. Each run shows:

- Which workflow fired
- The trigger conditions that matched
- What actions were executed
- Whether the run succeeded or failed
- When it triggered

## Templates tab

Templates are pre-built workflow recipes. Browse templates for common use cases and click to apply one as a starting point. You can customize trigger conditions and actions after applying a template.

<Tip>
  Start with a template and adjust the conditions. It's faster than building from scratch and helps you learn how workflows are structured.
</Tip>
