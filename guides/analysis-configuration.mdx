---
title: "Analysis configuration"
description: "Fine-tune how OpenBat analyzes your conversations — manage metadata fields, configure translation, and customize user and assistant analysis signals."
---

The Analysis Configuration page lets you control exactly how OpenBat processes your conversations. Manage metadata fields, enable translation, and customize what signals are extracted from user and assistant messages.

<Tabs>
  <Tab title="Metadata">
    ### Managed fields

    These are metadata keys you have explicitly chosen to track.

    | Column | Description |
    |--------|-------------|
    | **Key** | The metadata field name (e.g., `customer_plan`, `company_size`) |
    | **Type** | Data type: string, number, boolean, or enum |
    | **Status** | Whether the field is accepted or denied |
    | **Used** | How many conversations include this field |
    | **Discovered** | When the field was first seen |
    | **Actions** | Remove button to stop tracking |

    Accepted fields become available as filters in the conversations table and other data views.

    ### Discovered fields

    OpenBat automatically detects new metadata keys from incoming SDK data. The discovered fields table shows:

    - **Key** — The field name
    - **Sample value** — An example value from your data
    - **Occurrences** — How many conversations include this field
    - **+ Track** button — Click to start managing this field

    <Note>
      Discovered fields only appear when your SDK sends custom metadata that is not already being tracked. See the [SDK guide](/sdk/installation) for how to include custom metadata.
    </Note>
  </Tab>

  <Tab title="Translation">
    - **Enable translation** — Toggle to activate automatic translation of conversations
    - **Primary language** — Select the language conversations should be translated into

    When enabled, non-primary-language messages are translated automatically. Translated messages show a "Translated from [language]" badge in the conversation detail view.

    Translation is useful when your users speak multiple languages but your team reviews in one language.
  </Tab>

  <Tab title="User analysis">
    Configure which signals are extracted from **user messages**:

    - **Sentiment** — Score each user message as positive, neutral, or negative
    - **Intent** — Classify what the user is trying to do
    - **Topics** — Tag messages with relevant subject categories
    - **Flags** — Detect special signals like churn risk, buying signals, and capability gaps

    Each signal type can be toggled on or off. For intents and topics, you can define custom definitions:

    - **Slug** — Machine-readable identifier (e.g., `billing_inquiry`)
    - **Display name** — Human-readable label (e.g., "Billing inquiry")
    - **Description** — What this intent or topic means

    <Tip>
      Click **Generate Definitions** to let OpenBat suggest intent and topic definitions based on your existing conversation data. You can edit the suggestions before saving.
    </Tip>
  </Tab>

  <Tab title="Assistant analysis">
    Configure which signals are extracted from **assistant messages**:

    - **Behavior** — Detect how the bot responds: Hallucinating, Dodging, Verbose, Redirecting
    - **Outcome** — Classify conversation results: Resolved, Deflected, Partially Resolved
    - **Quality** — Score response quality and flag issues
    - **Flags** — Detect assistant-specific signals

    Each signal type can be toggled on or off independently.

    <Info>
      Disabling assistant analysis reduces your analysis costs but means you will not see behavior alerts or resolution outcomes on the [dashboard](/guides/dashboard).
    </Info>
  </Tab>

  <Tab title="Prompts">
    Customize the AI prompts used to analyze your conversations. This is an advanced feature for teams that want to fine-tune how analysis signals are extracted.

    <Warning>
      Changes to analysis prompts affect all future message analysis. Test changes on a small set of conversations before applying broadly.
    </Warning>
  </Tab>
</Tabs>
