The OpenBat SDK captures your chatbot conversations and sends them for analysis. Install once, and every conversation is automatically tracked with sentiment, behavior, and intent analysis.

## Getting your API key

<Steps>
  <Step title="Open your chatbot">
    Open your chatbot in the OpenBat dashboard.
  </Step>
  <Step title="Navigate to API keys">
    Go to **Settings** → **API Keys** tab.
  </Step>
  <Step title="Copy your key">
    Copy your API key (shown during onboarding, or view the prefix in settings).
  </Step>
</Steps>

<Tip>
  You can also copy your API key during the onboarding flow when you first create a chatbot.
</Tip>

## Quick overview

<CodeGroup>

```typescript TypeScript
import { OpenBat } from "@openbat/sdk";
const openbat = new OpenBat({ apiKey: "your-api-key" });
```

```python Python
from openbat import OpenBat
client = OpenBat(api_key="your-api-key")
```

```bash REST API
curl -X POST https://api.openbat.dev/v1/track \
  -H "Authorization: Bearer your-api-key"
```

</CodeGroup>

## Verifying your installation

After installing the SDK:

1. Send a test conversation through your chatbot
2. Open your chatbot dashboard in OpenBat
3. Check the **Conversations** page — your test conversation should appear within a few seconds
4. Click into the conversation to see analysis badges on each message

<Note>
  Analysis typically completes within seconds of receiving a conversation. If you don't see results, check that your API key is correct and that message analysis is enabled in [chatbot settings](/guides/settings).
</Note>

## What data to send

For the richest analytics, include this data with each conversation:

| Field | Required | Description |
|-------|----------|-------------|
| `conversationId` | Yes | Unique identifier for the conversation |
| `messages` | Yes | Array of message objects with `role` and `content` |
| `user.id` | Recommended | Unique user identifier |
| `user.email` | Recommended | User's email (enables user-level analytics) |
| `user.name` | Optional | Display name |
| `organization.id` | Recommended | Organization identifier |
| `organization.name` | Recommended | Organization name (enables org-level analytics) |
| Custom metadata | Optional | Any additional fields (e.g., `plan`, `mrr`, `industry`) |

<Info>
  Including `user` and `organization` data unlocks the [Users and Organizations](/guides/users-and-organizations) views and enables per-user sentiment tracking on the [dashboard](/guides/dashboard).
</Info>
