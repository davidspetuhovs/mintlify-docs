---

If you're building with the Vercel AI SDK, OpenBat integrates directly into your existing streaming setup. Track every conversation without changing your chat logic.

## Installation

```bash
npm install @openbat/sdk
```

## Setup

```typescript
import { openai } from "@ai-sdk/openai";
import { streamText } from "ai";
import { OpenBat } from "@openbat/sdk";

const openbat = new OpenBat({
  apiKey: process.env.OPENBAT_API_KEY,
});
```

## Tracking conversations

```typescript
export async function POST(req: Request) {
  const { messages, conversationId, user } = await req.json();

  const result = streamText({
    model: openai("gpt-4o"),
    messages,
  });

  // Track the conversation with OpenBat
  openbat.track({
    conversationId,
    messages,
    user: {
      id: user.id,
      email: user.email,
      name: user.name,
    },
    organization: {
      id: user.organizationId,
      name: user.organizationName,
    },
    metadata: {
      plan: user.plan,
      source: "web-chat",
    },
  });

  return result.toDataStreamResponse();
}
```

## Configuration options

| Option | Type | Description |
|--------|------|-------------|
| `apiKey` | `string` | Your OpenBat API key (required) |
| `baseUrl` | `string` | Custom API endpoint (optional, defaults to OpenBat cloud) |
| `debug` | `boolean` | Enable debug logging (optional) |

## Environment variables

<Tip>
  Store your API key in an environment variable. Add `OPENBAT_API_KEY` to your `.env.local` file and access it with `process.env.OPENBAT_API_KEY`.
</Tip>

```bash
# .env.local
OPENBAT_API_KEY=ob_live_your_key_here
```
