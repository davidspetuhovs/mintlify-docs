---
title: "Capture messages"
description: "POST /api/v1/capture — ingest conversation messages from your chatbot. The primary endpoint for SDK integration."
---

Send conversation messages to OpenBat for storage and analysis. This endpoint upserts conversations — it creates a new conversation if the `conversationId` is new, or appends messages and updates metadata if the conversation already exists.

## Endpoint

```
POST https://app.openbat.dev/api/v1/capture
Authorization: Bearer ob_live_...
Content-Type: application/json
```

## Request body

### Required fields

<ParamField body="conversationId" type="string" required>
  Your unique identifier for this conversation. Used to group messages together across multiple capture calls.
</ParamField>

<ParamField body="messages" type="array" required>
  An array of 1 to 100 message objects to ingest.

  <Expandable title="Message object properties">
    <ParamField body="role" type="string" required>
      The message author. One of `"user"`, `"assistant"`, or `"system"`.
    </ParamField>

    <ParamField body="content" type="string" required>
      The message text.
    </ParamField>

    <ParamField body="id" type="string">
      An external message ID. Used for deduplication — if you send the same `id` twice, the message is stored only once.
    </ParamField>

    <ParamField body="sentAt" type="string">
      ISO 8601 datetime indicating when the message was sent.
    </ParamField>
  </Expandable>
</ParamField>

### Optional fields

<ParamField body="user" type="object">
  Metadata about the end user in the conversation.

  <Expandable title="User object properties">
    <ParamField body="id" type="string">
      Your unique identifier for this user.
    </ParamField>

    <ParamField body="name" type="string">
      The user's display name.
    </ParamField>

    <ParamField body="email" type="string">
      The user's email address.
    </ParamField>

    <ParamField body="plan" type="string">
      The user's subscription plan (e.g. `"pro"`, `"enterprise"`).
    </ParamField>

    <ParamField body="createdAt" type="string">
      ISO 8601 datetime of when the user account was created.
    </ParamField>

    <ParamField body="industry" type="string">
      The user's industry.
    </ParamField>

    <ParamField body="mrr" type="number">
      Monthly recurring revenue associated with this user, in your base currency unit.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="organization" type="object">
  Metadata about the organization the user belongs to.

  <Expandable title="Organization object properties">
    <ParamField body="id" type="string">
      Your unique identifier for this organization.
    </ParamField>

    <ParamField body="name" type="string">
      The organization's name.
    </ParamField>

    <ParamField body="plan" type="string">
      The organization's subscription plan.
    </ParamField>

    <ParamField body="mrr" type="number">
      Monthly recurring revenue for this organization.
    </ParamField>

    <ParamField body="industry" type="string">
      The organization's industry.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="session" type="object">
  Metadata about the current browsing session.

  <Expandable title="Session object properties">
    <ParamField body="id" type="string">
      Your unique identifier for this session.
    </ParamField>

    <ParamField body="pageUrl" type="string">
      The URL of the page where the conversation is happening.
    </ParamField>

    <ParamField body="referrer" type="string">
      The referrer URL for this session.
    </ParamField>

    <ParamField body="device" type="string">
      The device type. One of `"desktop"`, `"mobile"`, or `"tablet"`.
    </ParamField>

    <ParamField body="country" type="string">
      The user's country.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="custom" type="Record<string, string | number | boolean>">
  Custom key-value fields. These are validated against the schema you define for your chatbot in the dashboard. Unknown fields produce a warning in the response but do not cause the request to fail.
</ParamField>

### Deprecated fields

  The following fields are still accepted but deprecated. Use the recommended replacements instead.
</Warning>

## Request example

```json
{
  "conversationId": "conv_abc123",
  "messages": [
    {
      "role": "user",
      "content": "How do I cancel my subscription?",
      "id": "msg_001",
      "sentAt": "2025-03-15T14:30:00Z"
    },
    {
      "role": "assistant",
      "content": "You can cancel your subscription from Settings > Billing. Would you like me to walk you through it?",
      "id": "msg_002",
      "sentAt": "2025-03-15T14:30:02Z"
    }
  ],
  "user": {
    "id": "user_456",
    "name": "Jane Smith",
    "email": "jane@example.com",
    "plan": "pro"
  },
  "session": {
    "pageUrl": "https://example.com/help",
    "device": "desktop"
  }
}
```

## Response

### Success (200 OK)

```json
{
  "ok": true,
  "conversationId": "d4f5a6b7-c8d9-4e0f-a1b2-c3d4e5f6a7b8",
  "messagesStored": 2,
  "analysisQueued": true,
  "warnings": []
}
```

<ResponseField name="ok" type="boolean">
  Always `true` on a successful response.
</ResponseField>

<ResponseField name="conversationId" type="string">
  OpenBat's internal UUID for the conversation.
</ResponseField>

<ResponseField name="messagesStored" type="number">
  The number of new messages inserted. Duplicate messages (matched by `id`) are excluded from this count.
</ResponseField>

<ResponseField name="analysisQueued" type="boolean">
  Whether LLM analysis (sentiment scoring, topic extraction) was queued for this batch of messages.
</ResponseField>

<ResponseField name="warnings" type="string[]">
  Non-fatal validation issues. For example, if you send a custom field that is not defined in your chatbot schema, it appears here.
</ResponseField>

### Response with warnings

```json
{
  "ok": true,
  "conversationId": "d4f5a6b7-c8d9-4e0f-a1b2-c3d4e5f6a7b8",
  "messagesStored": 2,
  "analysisQueued": true,
  "warnings": ["Custom field 'unknown_field' not in schema"]
}
```

### Error responses

| Status | Meaning |
|--------|---------|
| 400 | Invalid request body — missing required fields or wrong types |
| 401 | Missing or invalid API key |
| 429 | Rate limit exceeded |

## Behavior

<Note>
  The capture endpoint returns immediately. Sentiment analysis and other LLM processing run asynchronously after the response is sent.
</Note>

- **Upsert semantics** — if the `conversationId` has been seen before, new messages are appended and user/organization/session metadata is updated. Otherwise a new conversation is created.
- **Message deduplication** — messages are deduplicated by the `id` field. Sending the same message ID twice results in only one stored message.
- **Non-strict custom fields** — custom fields that are not defined in your chatbot's schema, or that have the wrong type, produce warnings but do not reject the request.
- **Message limit** — each request must contain between 1 and 100 messages.
